Case Study

Olibr —
Homepage Redesign

Redesigning a job platform homepage with a bolder visual direction, improved interaction patterns, and a more modern UI.

View Live ↗
Overview

The brief was simple.
The execution wasn't.

The existing Olibr homepage was plain, static, and forgettable — it didn't reflect the platform's AI-powered capabilities or give visitors a reason to stay. The brief: make it dynamic and bold.

I owned this end-to-end — from designing the hero section in Figma to directing the build through AI tools. Rather than just adding visuals, I rethought how the page could communicate Olibr's value through motion, layout, and interaction.

The entire codebase was generated and iterated using AI tools — with clear prompts, design-level direction, and hands-on tweaks to get each section to match the intended visual language. The decisions were mine; the execution was a collaboration with AI.

Process

How it came together.

01
Design — Figma

Started with the hero section in Figma — defining the visual direction, typography scale, and the bold, high-contrast aesthetic the page needed. Rough layout ideas for the features section were sketched here too.

02
Extend — AI-Assisted

Used AI tools to generate and iterate UI for sections beyond the Figma scope — feeding them clear prompts, reviewing output with a critical design eye, and refining until each section matched the visual language established in the hero.

03
Build — AI-Directed Code

The codebase was generated through AI tools in a React and Tailwind environment — directed with detailed prompts for each section, then tweaked at the design level to get spacing, motion, and visual details right.

The Work

The choices that made
the difference.

Hero Section
Bold typography. Motion from the start.

The hero was designed in Figma first — a clear departure from the original's flat layout. Large typographic hierarchy, an animated headline, and a strong visual anchor set the tone for the rest of the page before a user scrolls.

Features — Section 1
Scroll-stacked cards.

For the primary features section, I used a sticky scroll stack pattern — as you scroll, feature cards layer on top of each other with distinct colorful backgrounds. It turns a static list into an experience, and gives each feature the full viewport moment it deserves.

Features — Section 2
Auto-cycling with image on the right.

A second features section cycles through items automatically — each feature has a title and description on the left, with a corresponding product UI screenshot appearing on the right. Clean, purposeful, and requires no user effort to explore.

Problem / Solution
A toggle that tells the story.

Inspired by a pattern worth borrowing — a Problem/Solution toggle lets visitors flip between the pain of traditional hiring and Olibr's answer. It's a simple interaction that frames the product's value without a wall of text.
